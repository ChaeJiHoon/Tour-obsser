git 커밋 하는 법

파일을 수정하거나 새로운 파일을 추가한 후, 원격 저장소에 변경 사항을 올리려면 아래 단계를 따르면 됩니다. 이 과정은 기본적으로 다음과 같은 절차를 포함합니다:

1. 파일 수정 및 추가:
   파일을 수정하거나 새 파일을 프로젝트 폴더에 추가합니다.

2. 변경 사항 스테이징:
   변경된 파일을 Git의 스테이징 영역에 추가합니다.

   ```   git add .
   ```

   `.`은 현재 디렉토리의 모든 변경된 파일을 스테이징하는 명령어입니다. 특정 파일만 추가하려면 `git add <파일이름>`을 사용하세요.

3. 커밋:
   변경 사항을 커밋합니다. 커밋 메시지는 변경된 내용을 설명하는 메시지입니다.

   ```
   git commit -m "Your commit message describing the changes"
   ```

4. 푸시:
   로컬 브랜치의 커밋을 원격 저장소로 푸시합니다.

   ```
   git push origin <브랜치이름>
   ```

   여기서 `<브랜치이름>`은 현재 작업 중인 브랜치의 이름입니다. 예를 들어, `tjsdud` 브랜치에서 작업 중이라면 `git push origin tjsdud`를 사용합니다.

 예시

1. 파일 수정 및 추가:
   예를 들어, `example.txt` 파일을 수정하거나 추가합니다.

2. 변경 사항 스테이징:
   ```
   git add example.txt
   ```

3. 커밋:
   ```
   git commit -m "Update example.txt with new information"
   ```

4. 푸시:
   ```
   git push origin tjsdud
   ```

이 단계를 통해 파일을 수정하고 원격 저장소에 변경 사항을 올릴 수 있습니다.
